# MoM Suite

Minutes of Meeting system built with:

- `MoM.Web`: ASP.NET Core MVC frontend
- `MoM.Api`: ASP.NET Core Web API backend

## Features

- Welcome page and dashboard flow
- Create, edit, preview, delete MOM records
- Track attendees by name with present / absent state
- Export MOM as PDF using QuestPDF
- Dashboard analytics for meetings and attendance
- Scalar API reference UI

## Project Structure

- `MoM.Web` : frontend MVC app
- `MoM.Api` : backend API, EF Core, PDF generation
- `MoM.sln` : full solution
- `MoM.Web/MoM.Web.sln` : frontend-only solution

## Default Local Ports

- Frontend: `http://localhost:5156`
- Backend API: `http://localhost:5157`
- Scalar UI: `http://localhost:5157/scalar`

## Backend Setup

Go to the API project:

```powershell
cd .\MoM.Api
```

Create and apply migrations:

```powershell
dotnet ef migrations add InitialCreate --project .\MoM.Api.csproj --startup-project .\MoM.Api.csproj
dotnet ef database update --project .\MoM.Api.csproj --startup-project .\MoM.Api.csproj
```

If you already created old tables manually or from a previous incompatible schema, delete the old `MoM_DB` database first, then run the migration commands again.

Run the backend:

```powershell
dotnet run
```

## Frontend Setup

Go to the frontend project:

```powershell
cd .\MoM.Web
dotnet run
```

Open:

```text
http://localhost:5156
```

## Workflow

1. Open the welcome page
2. Go to dashboard
3. Create MOM
4. Add meeting details, meeting type, attendees, agenda, and action items
5. Save the MOM
6. Preview, edit, or export PDF from the dashboard

## Notes

- Frontend calls the backend through an MVC proxy controller
- PDF export is generated by `QuestPDF`
- API documentation is exposed through `Scalar.AspNetCore`
- Attendance statistics are computed from attendee rows, not manual counts
